# 快速上手（新手指南）

这份指南假设你是第一次接触这个工具，从零开始一步步走通：**装好 → 启动 → 把图入库 → 选词组合 → 生成新图**。

如果中途卡住，先看本页最后的「常见卡点」，再去翻 [RUNBOOK](RUNBOOK.md)。

---

## 0. 这个工具到底是干什么的

一句话：**把你喜欢的图片"拆解"成一条条可复用的中文视觉词条，攒成一个本地资料库；之后跨多张图勾选词条、组合成提示词，生成一张全新的原创图。**

它**不是**用来仿照或复刻原图的——全程都在刻意避免照抄原图，只迁移"视觉语言"（构图、色彩、材质、情绪这类）。

整个流程分两个阶段：

- **阶段一·入库**：需要一个能看图的 AI agent（Claude Code、Codex 或其他）帮你分析图片，把结果写进本地资料库。
- **阶段二·选词生图**：在本地网页里点选词条、组合提示词、生成图片。这一步**不需要 agent**，自己在网页上点就行。

## 1. 先准备这些

| 必须 | 说明 |
|---|---|
| Node.js 20+ | 跑本地服务。去 https://nodejs.org/ 下载安装。装完在终端输 `node -v` 能看到版本号即可。 |
| Git | 一键安装脚本会用到。Windows: https://git-scm.com/download/win |

| 按需 | 说明 |
|---|---|

| 一个图片生成 API Key | 调 `gpt-image-2` 生图要用。可以是 OpenAI 官方的，也可以是兼容端点的。只想攒资料库、暂时不生图的话，可以先不准备。 |

## 2. 安装

### 方式 A：一键安装（图省事）

Windows（PowerShell）：

```powershell
irm https://raw.githubusercontent.com/LioraRndr/ArtworkDestructionTool/main/install.ps1 | iex
```

macOS / Linux：

```bash
curl -fsSL https://raw.githubusercontent.com/LioraRndr/ArtworkDestructionTool/main/install.sh | bash
```

> 这条命令会从网上下载脚本并直接执行。如果你对"下载即执行"有顾虑，用下面的方式 B，可以先看清楚每一步再跑。

### 方式 B：手动安装（更透明，推荐想看清楚的人）

```bash
git clone https://github.com/LioraRndr/ArtworkDestructionTool.git
cd ArtworkDestructionTool
npm start
```

手动方式不会自动装 `gpt-image` CLI、不会复制随包 skill、也不会建桌面快捷方式。这些都可以等真正用到阶段二时再补。

安装脚本（方式 A）会额外帮你做：建 `data/` 数据目录、把 `artwork-destruction-tool` skill 装到 `~/.agents/skills/`、尝试装 `gpt-image` CLI、Windows 上建桌面快捷方式。

## 3. 启动并打开页面

在项目目录里：

```powershell
npm start
```

Windows 用方式 A 装的，也可以直接双击桌面的 **Artwork Destruction Tool** 快捷方式。

然后浏览器打开：

```text
http://localhost:5173/
```

看到页面就说明服务起来了。第一次打开资料库是空的，很正常。

## 4. 阶段一：把图片"拆"进资料库

目标：让你的 agent 分析图片，把结构化结果 POST 到 `http://localhost:5173/api/records`，前端就会自动刷新出来。

### 4.1 准备指令

页面上有一段现成的"交接指令"（handoff prompt），点页面里的复制按钮把它复制下来，连同图片一起发给你的 agent。这段指令已经写好了所有要求（分析、转 JSON、POST 地址、不许照抄原图等）。

前提：你的 agent 那边已经装好了随包的 `artwork-destruction-tool` skill（用方式 A 安装时会自动装到 `~/.agents/skills/`）。

### 4.2 ⚠️ 给 agent 图片时：用"拖拽文件"，别用"粘贴"

这一点很关键，很多人在这里踩坑：

- ✅ **把图片文件拖拽进 agent**：agent 能读到本地文件，原图会一起进资料库，前端能看到预览。
- ❌ **直接粘贴图片进对话**：图片没有落地成文件，agent 读不到它，结果资料库里这条记录**没有原图**，前端显示"无图"。词条还是能正常解构，但你看不到原图对照。

如果你已经用粘贴的方式入库了、发现没有原图，别急——见下面 4.4 的"手动补传原图"。

### 4.3 确认入库成功

agent POST 成功后，前端**每 3 秒自动刷新**，你会在左侧资料库列表看到新卡片。点开就能看到总览、基础信息、风格/情绪标签和分类词条。

### 4.4 没有原图？手动补传

如果某条资料显示"无图 / 暂无原图"，可以在前端手动补：

1. 在左侧资料库点选那张资料。
2. 在"当前原图"区域，**三选一**：
   - 点「上传 / 替换原图」选一个图片文件；
   - 把图片**拖拽**到左侧预览框；
   - 光标不在输入框时，直接 **Ctrl+V 粘贴**。
3. 补传只更新这条记录的原图，已经解构好的词条不受影响。

## 5. 阶段二：选词、组合提示词、生成新图

这一步纯在网页上操作，不需要 agent。

1. **选词条**：在资料库里点开一张或多张资料，勾选你想要的词条（可以跨不同图片混选）。
2. **填主题（可留空）**：比如"未来城市海报"。留空的话，会让模型基于选中的视觉要素自己构思一个全新主题。
3. **选模板**：自由融合 / 海报封面 / 品牌视觉 / 游戏角色概念。
4. **点「组合选中要素」**：自动把所有勾选的词条拼成一段完整的中文提示词，显示在文本框里。这段提示词里已经内置了"不照抄原图"的限制。
5. **填生成设置**：
   - **API Key**：在输入框填，或事先用环境变量 `OPENAI_API_KEY`（更安全，见下）。
   - **Base URL**：用官方端点可留空；用第三方兼容端点通常要以 `/v1` 结尾。
   - **尺寸**：选比例预设，或填自定义像素（宽高都要是 16 的倍数，最长边 ≤ 3840，宽高比 ≤ 3:1）。
   - 模型固定是 `gpt-image-2`，不用选。
6. **点「生成」**：稍等，右侧会显示生成结果，历史也会留档。

> 关于 API Key 的安全：填在前端的 Key 会**明文存在浏览器里**方便下次免填。公用电脑别这么干，改用环境变量 `OPENAI_API_KEY` 更稳妥。详见 [RUNBOOK · API Key 的存储与安全提示](RUNBOOK.md)。

## 6. 常见卡点速查

| 现象 | 多半是因为 | 怎么办 |
|---|---|---|
| 资料库一直空的 | agent 没 POST 成功，或服务没起来 | 确认 `npm start` 在跑、地址是 `http://localhost:5173`；让 agent 重发 POST |
| 资料有词条但"无图" | 入库时用了粘贴而非拖拽文件 | 见 [4.4 手动补传原图](#44-没有原图手动补传) |
| 生成报 HTTP 405 | Base URL 不对，或端点不支持该接口 | 关掉"上传资料库原图作为参考图"；确认 Base URL 以 `/v1` 结尾；详见 RUNBOOK |
| 生成失败原因不明 | — | 看右侧生成预览区的 `stderr` / `stdout` 诊断，不要只看顶部状态栏 |
| 改了 `server.js` 没生效 | 服务端代码要重启 | 重启 `npm start`（前端代码/样式刷新即可，服务端必须重启） |

更全的排错在 [RUNBOOK](RUNBOOK.md)；接口细节和尺寸规则在 [API](API.md)。

## 7. 一次完整走通（示例）

1. `npm start`，浏览器打开 `http://localhost:5173/`。
2. 复制页面上的交接指令，**拖拽**一张你喜欢的海报图给 Claude Code/Codex，连同指令发过去。
3. agent 分析完 POST 入库，几秒后左侧出现新卡片，点开能看到原图和一堆中文词条。
4. 重复第 2~3 步，再入库两三张风格不同的图。
5. 跨这几张图勾选你最想要的词条（比如 A 图的配色 + B 图的构图 + C 图的材质）。
6. 主题填"赛博城市音乐节主视觉"，模板选"海报 / 封面"，点「组合选中要素」。
7. 填好 API Key 和 Base URL，尺寸选 `16:9`，点「生成」。
8. 右侧出图，满意就留着，不满意调词条或主题再生成一次。
